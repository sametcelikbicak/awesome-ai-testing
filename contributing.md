# Contribution Guidelines

Thanks for considering a contribution.

Please ensure your pull request follows these rules:

- The tool, framework, or resource should genuinely use AI/ML/LLMs as a core feature, not as a marketing tagline.
- Use the format: `- [Name](link) BADGE - Description.` where BADGE is 🆓, 💰, or 🆓💰.
- Keep descriptions concise (one sentence preferred, two maximum).
- Place the entry in the most appropriate category. If unsure, suggest a new category in your PR.
- Within a category, entries are roughly ordered by popularity or relevance, not strictly alphabetical.
- Make an individual pull request for each suggestion.
- Check for typos and broken links before submitting.
- The suggested addition should not be a duplicate.
- New categories or improvements to the existing categorization are welcome.

## Pull Request Title Format

Use one of these prefixes:
- `Add: [Tool Name]` for new entries
- `Update: [Tool Name]` for description changes
- `Fix: [issue]` for typos or broken links
- `Remove: [Tool Name]` for entries that should no longer be included

## What I won't merge

- Tools that have been abandoned (no commits in 2+ years) unless historically significant
- Pure marketing landing pages without a real product
- Duplicate entries (search the README first)
- Entries with no clear AI/ML component

If you maintain a tool listed here and the description is inaccurate, please open a PR or issue to correct it.
